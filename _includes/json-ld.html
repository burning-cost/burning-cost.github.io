{% comment %} json-ld.html ------------ Emits a Schema.org Article JSON-LD block for individual blog posts. Included from _layouts/post.html inside . We use both BlogPosting and TechArticle @type values. Google accepts arrays here and TechArticle is appropriate for the technical audience. Using both maximises signal without any downside. All string values pass through | jsonify to handle apostrophes, quotes, and other characters that would otherwise break JSON. Date fields use date_to_xmlschema to produce ISO 8601 format as required by Schema.org. articleSection: first category if set (e.g. "pricing", "tutorials"). keywords: all tags joined as a comma-separated string. {% endcomment %}
